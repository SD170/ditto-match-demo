# Future Us Stitch Design Prompts

Project: Stitch project `projects/9138456706527534014` / `Modern Landing Page Design`

Design direction: extend the current Ditto AI dark, cinematic, neon-magenta dating app style. The existing repo uses charcoal/black surfaces, primary pink `#F748B1`, cyan accents, large compressed editorial headlines, rounded image cards, glassy panels, noisy glow backgrounds, and mobile-first but desktop-polished layouts.

## Screen 1: Future Us Entry Modal

Create a desktop web app screen for Ditto AI called “Future Us Entry Modal”. It should appear immediately after the existing “It’s a match” celebration screen. Keep the same dark charcoal background, neon pink primary accent, cyan secondary accent, rounded cards, glow lighting, and Gen Z editorial dating-app energy.

Screen content:
- Full-screen dark overlay with subtle magenta/cyan blurred orbs.
- Centered modal/card with two user profile images partially overlapping, like two agents being paired.
- Big headline: “Preview Future Us”
- Subtitle: “Your private dating agents can simulate this connection before Ditto sends the Wednesday invite.”
- A small privacy pill: “Private simulation · Not shown to your match”
- A confidence preview row: “5 signal sources connected · 82% context confidence”
- Mini connected signal chips: AI Memory, Maps, Calendar, Food, Music. Connected chips glow pink/cyan; disconnected chips are dim.
- Primary CTA: “Run Future Us”
- Secondary CTA: “Skip and send date invite”
- Small science note at bottom: “Built around relationship-specific signals: rhythm, responsiveness, appreciation, conflict repair, and logistics.”

Visual layout:
- Preserve the existing Ditto match modal DNA: dramatic centered card, expressive typography, playful stickers/arrows but cleaner and more premium.
- Use a glassmorphic central panel with no hard borders, only glow and tonal layering.
- Include tiny agent-node lines between the two profile images.
- Make the CTA impossible to miss.

## Screen 2: Future Us Simulation Dashboard

Create a desktop web app screen for Ditto AI called “Future Us Simulation Dashboard”. This is the main result screen after the simulation runs. It should feel like a futuristic dating mission-control report, but still romantic and user-friendly.

Core concept:
Ditto uses opt-in persona signals to simulate how a match may feel in real life. The simulation is not a prediction; it surfaces conditions that give the first connection the best chance.

Screen content:
- Header/nav consistent with the existing Ditto app.
- Left vertical step rail: “03 · Future Us”
- Top hero section:
  - Brow label: “Agentic Relationship Simulation”
  - Huge headline: “Future Us”
  - Subtitle: “Your private agents simulated chemistry, friction, repair, and real-world logistics.”
  - Couple title card: “The Slow-Burn Campus Explorers”
  - Summary: “Best path: a low-pressure Wednesday coffee walk. Main spark: quiet curiosity. Main risk: both hide direct interest behind humor.”
- Right-side “Context Confidence” card:
  - Circular or horizontal meter at 82%
  - Explanation: “High context: AI Memory, Maps, Calendar, Food, and Music connected.”
- Connected Signal Cards row:
  - AI Memory +22
  - Maps +16
  - Calendar +14
  - Food +7
  - Music +8
  - Messages locked/private
- Chemistry Map section with 6 horizontal bars or radar-style cards:
  - Rhythm Fit 91
  - Emotional Safety 78
  - Novelty + Growth 86
  - Conflict Repair 64
  - Intention Alignment 83
  - Practical Logistics 88
- Bottom tabs/cards preview:
  - First Date
  - Ordinary Tuesday
  - First Misread
  - Repair Moment
  - Friends Test

Visual style:
- Dark cinematic background with grid/noise texture.
- Magenta and cyan glow accents.
- Large rounded panels, no thin harsh borders.
- Use profile images as small circular avatars near the title.
- Data should feel warm, not enterprise-y.
- The user should instantly understand: this is deeper than a match score.

## Screen 3: Future Us Scenario Simulator

Create a desktop web app screen for Ditto AI called “Future Us Scenario Simulator”. This screen shows the scenario-by-scenario relationship simulation.

Screen content:
- Header: “Scenario Simulator”
- Subtitle: “Not a prediction. A map of where this connection could spark, stall, or grow.”
- A horizontal scenario tab bar:
  - Natural Overlap
  - First Date
  - Ordinary Tuesday
  - First Misread
  - Repair Moment
  - Friends Test
  - Growth Arc
- Main active scenario: “Natural Overlap”
  - Title: “You already live near each other’s orbit.”
  - Scene: “Both of you cross North Quad between 7:00–8:00 PM on Wednesdays. Ditto should not manufacture a dramatic meet-cute; it should convert a real overlap into an intentional, low-pressure first date.”
  - Evidence chips: “Maps overlap”, “Calendar window”, “Evening energy”, “Coffee preference”
  - Best move: “Meet at Prism Coffee Atrium at 7:15 PM, then walk toward the Art Library.”
- Side panel: “Why this matters”
  - Rhythm fit reduces planning friction.
  - Low-pressure settings help both personas open up.
  - Relationship-specific signals matter more than static traits.
- Below, show mini cards for other scenarios:
  - First Misread: “Slow replies during deadline week may look like low interest.”
  - Repair Moment: “Direct reassurance beats more texting.”
  - Growth Arc: “She adds spontaneity; he adds consistency.”

Visual style:
- Like a futuristic relationship lab, but romantic.
- Use animated-looking connection lines, map path hints, small calendar blocks, and agent nodes.
- Keep it aligned with the existing Ditto dark neon UI.

## Screen 4: Future Us Best Date Compiler

Create a desktop web app screen for Ditto AI called “Best Date Compiler”. This screen turns the Future Us simulation into Ditto’s concrete real-world date plan.

Screen content:
- Header: “Best Date Compiler”
- Main recommendation card:
  - “Wednesday 7:15 PM · Prism Coffee Atrium”
  - Plan: “Grab matcha or chai, then take a 25-minute walk toward the Art Library. Hard stop at 8:05 so it feels safe, low-pressure, and easy to extend if chemistry is real.”
  - CTA: “Send Wednesday Invite”
- Why this date works section with 5 reason cards:
  - Schedule overlap: “Both free after 7 PM.”
  - Place overlap: “Both pass North Quad weekly.”
  - Food signal: “Both prefer warm drinks / casual food over loud bars.”
  - Emotional pace: “Both open up better while walking than sitting face-to-face.”
  - Safety: “Public, campus-safe, short default duration.”
- Watchouts + Repair Moves section:
  - Watchout: “Do not interpret deadline-week slow replies as disinterest.”
  - Repair move: “Say the direct thing early: ‘I’m interested, just overloaded this week.’”
  - Watchout: “Both may use humor to dodge vulnerability.”
  - Repair move: “Ask one sincere question before the date ends.”
- Footer note:
  - “Future Us is a private simulation based on opt-in context. It is not a prediction or compatibility guarantee.”

Visual style:
- The date plan card should feel like a premium ticket/invite.
- Use neon pink CTA, cyan secondary highlights, rounded cards, big typography, and dark glass surfaces.
- Make it feel like Ditto has compiled a real date, not just generated advice.
